SyntaxNode

getUse()->String abstract

returns the purpose of the information the node stores for the syntax tree
options are:

  • "Identifier": a single unbroken word in the code
  • "Literal": a single number or string in the code
  • "Operator": 1, 2, or several nodes joined by specially defined operator symbols (see Operator)
  • "Group": a parenthesis surrounded node
  • "Chain": 2 nodes joined by a space
  • "Control": a control statement (condition node + body node + else/nelse condition and/or body nodes)

getName()->String abstract

returns the value of the information the node stores for the syntax tree
ie if the node is a control, this would be the name of the control statement
ie if the node is an identifier, this would be the name of the word

getSize()->int abstract

returns the number of children nodes stored by the node

getParent()->SyntaxNode

returns the current parent of the node

getParent(SyntaxNode)

sets the current parent of the node

addTo(FunctionalNode) abstract

adds this node as a functional node (perform analysis and evaluation)

isFunctionCall()

returns whether the format of the node is that of a function header; false by default
the format for a function header is name(arguments)

isFunctionSignature()

returns whether the format of the node is that of a function signature; false by default
the format for a function signature is name(arguments) or name{returns}(arguments)
